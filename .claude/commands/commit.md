コミットメッセージを自動生成してコミットを行う。

## 手順

1. まず、ユーザーに「ステージされた変更のみをコミットしますか？それとも全ての変更をコミットしますか？」と質問する
   - 選択肢: 「ステージ済みのみ」「全ての変更」

2. `git status` と `git diff`（または `git diff --staged`）を実行して変更内容を確認する

3. `git log --oneline -5` で最近のコミットメッセージのスタイルを確認する

4. 変更内容を分析し、Conventional Commits形式でメッセージを生成する

5. コミットを実行する:
   - 「全ての変更」の場合: 変更されたファイルを個別に `git add` してからコミット
   - 「ステージ済みのみ」の場合: そのままコミット

6. コミット後、`git status` で結果を表示する

## 注意事項

- `git add -A` や `git add .` は使用しない（機密ファイルの誤コミット防止）
- コミットメッセージは Co-Authored-By を含める
